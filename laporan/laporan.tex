\section{Latar Belakang}
Information Retrieval (IR) adalah proses pengambilan dokumen yang relevan dari koleksi besar berdasarkan query pengguna. Sistem IR konvensional menggunakan pencocokan kata kunci yang sering kali gagal menangkap makna semantik dari query.

Word2Vec adalah model embedding kata yang merepresentasikan kata dalam ruang vektor berdimensi tinggi, di mana kata-kata dengan makna serupa berada berdekatan. Dengan memanfaatkan Word2Vec untuk query expansion, sistem dapat menambahkan kata-kata semantik yang relevan ke query awal, sehingga meningkatkan recall tanpa mengorbankan precision secara signifikan.

Proyek ini mengimplementasikan sistem IR dengan fitur query expansion berbasis Word2Vec menggunakan dataset CISI (Collection of Scientific Information) yang berisi 1460 dokumen dan 112 query uji.

\section{Tujuan}
\begin{enumerate}
    \item Mengimplementasikan sistem IR berbasis TF-IDF cosine similarity dengan 7 skema pembobotan
    \item Mengintegrasikan Word2Vec untuk semantic query expansion
    \item Mengevaluasi performa sistem menggunakan Mean Average Precision (MAP)
    \item Membandingkan hasil retrieval antara query original dan query expanded
    \item Membangun REST API dengan caching untuk performa optimal
    \item Menyediakan antarmuka web interaktif dengan dark mode
\end{enumerate}

\section{Ruang Lingkup}
\begin{itemize}
    \item Dataset: CISI collection (1460 dokumen, 112 query, 76 query dengan qrels)
    \item Preprocessing: Lowercasing, tokenization, stopword removal, Porter stemming
    \item Ranking: TF-IDF dengan 7 skema pembobotan (tf\_raw, tf\_log, tf\_bin, tf\_aug, idf, tfidf, tfidf\_cos)
    \item Query Expansion: Word2Vec Google News 300 dimensions
    \item Evaluasi: Mean Average Precision (MAP) dengan caching 28 variasi konfigurasi
    \item Backend: FastAPI dengan sistem cache (index dan MAP)
    \item Frontend: React + Vite + TypeScript + TailwindCSS
\end{itemize}

\section{Modul Program}

\subsection{preprocessor.py}
Modul ini menangani preprocessing teks dengan fungsi-fungsi:
\begin{itemize}
    \item \texttt{preprocess(text, stemming, remove\_stopword)}\\ - Fungsi utama preprocessing
    \item \texttt{\_normalize\_text(text)}\\ - Lowercase dan hapus karakter non-alfanumerik menggunakan regex
    \item \texttt{\_tokenize(text)} - Split teks menjadi token berdasarkan spasi
    \item \texttt{\_remove\_stopwords(tokens)} - Hapus stopwords English dari NLTK corpus
    \item \texttt{\_stem\_tokens(tokens)} - Porter Stemmer dari NLTK
\end{itemize}

\subsection{parser.py}
Parser untuk format dokumen CISI:
\begin{itemize}
    \item \texttt{parse\_docs(file\_path)} - Parse dokumen dari file
    \item \texttt{parse\_text(raw)} - Parse dokumen dari string
    \item Tags yang didukung: \texttt{.I} (ID), \texttt{.T} (Title), \texttt{.A} (Author), \\ \texttt{.W} (Abstract), \texttt{.B} (Bibliography)
    \item Normalisasi doc\_id ke format \texttt{DOC\#\#\#}
    \item Output: \texttt{dict[doc\_id, content]}
\end{itemize}

\subsection{indexer.py}
Pembangunan inverted index:
\begin{itemize}
    \item \texttt{build\_index(docs, stemming, remove\_stopword)} - Build inverted index
    \item Struktur index: \texttt{dict[term, \{"df": int, "postings": \{doc\_id: tf\}\}]}
    \item \texttt{save\_index(index, file\_path)} - Simpan index ke JSON atau Pickle
    \item \texttt{load\_index(file\_path)} - Load index dari file
    \item \texttt{get\_document\_terms(index, doc\_id)} - Get terms untuk dokumen tertentu
\end{itemize}

\subsection{weighting.py}
Modul pembobotan term dengan 7 skema:
\begin{itemize}
    \item \texttt{get\_weight(term, doc\_id, scheme, index)}\\ - Hitung bobot term untuk dokumen
    \item \texttt{get\_n\_docs(index)} - Hitung jumlah dokumen dalam koleksi
    \item Skema yang didukung:
    \begin{enumerate}
        \item \textbf{tf\_raw} - Term frequency mentah
        \item \textbf{tf\_log} - $1 + \log(tf)$
        \item \textbf{tf\_bin} - Binary (1 jika ada, 0 jika tidak)
        \item \textbf{tf\_aug} - $0.5 + 0.5 \times \frac{tf}{max\_tf}$
        \item \textbf{idf} - $\log(\frac{N}{df})$
        \item \textbf{tfidf} - $(1 + \log(tf)) \times \log(\frac{N}{df})$
        \item \textbf{tfidf\_cos} - TF-IDF dengan cosine normalization
    \end{enumerate}
\end{itemize}

\subsection{retrieval.py}
Engine untuk ranking dokumen:
\begin{itemize}
    \item \texttt{rank\_documents(query\_terms, index, scheme)}\\ - Ranking dokumen berdasarkan similarity
    \item \texttt{compute\_query\_weight(term, tf, max\_tf, n\_docs, df, scheme)}\\ - Hitung bobot query term
    \item Menggunakan \texttt{weighting.py} untuk perhitungan bobot
    \item Untuk \texttt{tfidf\_cos}: melakukan cosine normalization pada query vector
    \item Output: \texttt{list[(doc\_id, score)]} diurutkan descending
\end{itemize}

\subsection{word2vec\_engine.py}
Wrapper untuk model Word2Vec:
\begin{itemize}
    \item \texttt{init\_engine()} - Inisialisasi singleton engine
    \item \texttt{get\_engine()} - Get engine instance
    \item \texttt{Word2VecEngine.get\_similar(term, top\_n)} - Get semantically similar terms dengan 5 filter:
    \begin{enumerate}
        \item Lowercase only (buang proper noun)
        \item Buang kata dengan underscore
        \item Buang kata terlalu pendek ($< 3$ karakter)
        \item Buang kata yang terlalu mirip dengan input (substring match)
        \item Buang kata terlalu panjang ($> 20$ karakter)
    \end{enumerate}
    \item \texttt{Word2VecEngine.similarity(term1, term2)} - Cosine similarity dua kata
    \item \texttt{Word2VecEngine.train()} - Train model dari koleksi dokumen
    \item \texttt{save\_disk\_cache()} - Simpan cache ke disk
\end{itemize}

\subsection{query\_expansion.py}
Logika query expansion:
\begin{itemize}
    \item \texttt{expand\_query(query\_terms, top\_n)} - Expand dengan batas term
    \item \texttt{expand\_query\_all(query\_terms)} - Expand tanpa batas (top-50 similar per term)
    \item \texttt{get\_expanded\_query\_terms(query\_terms, top\_n, expand\_all)}\\ - Combine original + expanded
    \item \texttt{format\_expansion\_detail(query\_terms, expanded, query\_id)}\\ - Format output untuk logging
\end{itemize}

\subsection{evaluator.py}
Evaluasi performa sistem:
\begin{itemize}
    \item \texttt{parse\_qrels(file\_path)}\\ - Parse relevance judgments (auto-detect format: binary vs graded)
    \item \texttt{parse\_queries(file\_path)} - Parse queries dari format CISI
    \item \texttt{compute\_ap(rankings, relevant\_docs)} - Average Precision per query
    \item \texttt{compute\_map(rankings, qrels)} - Mean Average Precision overall
    \item \texttt{run\_experiment()} - Run full experiment dengan support untuk caching (docs dan inverted\_index dari caller)
    \item \texttt{format\_report(results)} / \texttt{write\_report(results)}\\ - Generate laporan evaluasi
\end{itemize}

\subsection{api/main.py (Backend)}
FastAPI REST API dengan fitur caching:
\begin{itemize}
    \item \textbf{SystemState} - Global state untuk cache: docs, indices, map\_cache
    \item \textbf{MAP Cache Persistence} - Simpan/load dari \texttt{output/map\_cache.pkl}
    \item \textbf{Index Cache} - Simpan/load dari \texttt{output/index\_\{stemming\}\_\{stopword\}.pkl}
    \item \textbf{Benchmark Loop} - Pre-calculate MAP untuk 28 variasi saat startup
    \item \textbf{Endpoints:}
    \begin{itemize}
        \item \texttt{POST /api/search} - Search dengan query expansion
        \item \texttt{POST /api/expand} - Preview expansion terms
        \item \texttt{GET /api/index/\{doc\_id\}} - Get inverted index untuk dokumen
        \item \texttt{GET /api/map} - Get MAP evaluation report
        \item \texttt{POST /api/index/rebuild} - Hapus semua index dan cache
        \item \texttt{POST /api/map/recalculate/single}\\ - Recalculate MAP untuk 1 konfigurasi
        \item \texttt{POST /api/map/recalculate/all} - Recalculate MAP untuk seluruh 28 variasi
    \end{itemize}
\end{itemize}

\section{Alur Program}
\begin{enumerate}
    \item \textbf{Startup}: Muat model Word2Vec, parse dokumen CISI, muat cache MAP, jalankan benchmark loop 28 variasi
    \item \textbf{Search Request}: Terima request, dapatkan/buat index cache, preprocess query, expand query, rank dokumen, get MAP dari cache, return response
    \item \textbf{Evaluation}: Untuk setiap query di qrels, preprocess, retrieve, compute AP, compute MAP
\end{enumerate}

\section{Setup Eksperimen}
\begin{itemize}

    \item \textbf{Jumlah Dokumen}: 1460

\end{itemize}

\section{Metrik Evaluasi}
\subsection{Average Precision (AP)}
Average Precision mengukur kualitas ranking untuk satu query:
\begin{equation}
    AP = \frac{1}{|R|} \sum_{k=1}^{n} P(k) \times rel(k)
\end{equation}
dimana $|R|$ = jumlah dokumen relevan, $P(k)$ = precision pada posisi ke-$k$, $rel(k)$ = 1 jika dokumen ke-$k$ relevan.

\subsection{Mean Average Precision (MAP)}
Mean Average Precision adalah rata-rata AP dari semua query:
\begin{equation}
    MAP = \frac{1}{|Q|} \sum_{q=1}^{|Q|} AP(q)
\end{equation}
dimana $|Q|$ = jumlah query.

Tiga konfigurasi lainnya mengalami penurunan, dengan penurunan terbesar terjadi pada konfigurasi tanpa stemming dan tanpa stopword removal.

\subsection{Query dengan Peningkatan AP Signifikan}

Query 111 juga mencatat peningkatan tertinggi (+0.0440), menunjukkan bahwa expansion mampu menangkap variasi terminologi dari topik yang sama.

\section{Analisis Perbandingan MAP Original vs Expanded Query}

\subsection{Pengaruh Query Expansion terhadap MAP}
Query expansion menggunakan Word2Vec bertujuan untuk menambahkan term-term yang secara semantik mirip dengan term original query. Secara teoritis, ini meningkatkan recall karena sistem dapat menemukan dokumen yang tidak mengandung term original tetapi mengandung term semantik yang relevan.

Berdasarkan hasil eksperimen pada Tabel~\ref{tab:map_overall}, query expansion menunjukkan dampak yang bervariasi terhadap MAP bergantung pada konfigurasi preprocessing:
\begin{itemize}
    \item Pada konfigurasi \textbf{stemming=True, stopword=True}, MAP meningkat sebesar \textbf{+6.55\%} (dari 0.1680 menjadi 0.1790). Ini adalah satu-satunya konfigurasi yang menunjukkan peningkatan setelah expansion.
    \item Pada konfigurasi \textbf{stemming=True, stopword=False}, MAP menurun sebesar \textbf{-6.93\%} (dari 0.1746 menjadi 0.1625).
    \item Pada konfigurasi \textbf{stemming=False, stopword=True}, MAP menurun sebesar \textbf{-12.34\%} (dari 0.1661 menjadi 0.1456).
    \item Pada konfigurasi \textbf{stemming=False, stopword=False}, MAP menurun sebesar \textbf{-11.26\%} (dari 0.1616 menjadi 0.1434).
\end{itemize}
Secara keseluruhan, query expansion hanya memberikan perbaikan ketika stemming dan stopword removal diterapkan bersamaan. Hal ini menunjukkan bahwa tanpa normalisasi (stemming) dan pembersihan noise (stopword), term-term expansion justru menambah distraksi yang menurunkan performa retrieval.

\subsection{Query yang Mengalami Peningkatan Signifikan}
Query dengan term yang memiliki representasi Word2Vec yang baik (kata-kata umum dalam vocabulary Google News) cenderung beneficia dari expansion.

Berdasarkan Tabel~\ref{tab:ap_increase}, tiga query yang mengalami peningkatan AP signifikan adalah:
\begin{itemize}
    \item \textbf{Query 111} (+0.0440, tanpa stemming \& stopword): AP naik dari 0.3362 ke 0.3802. Term dalam query ini kemungkinan besar adalah kata-kata umum yang memiliki banyak tetangga semantik relevan di ruang vektor Word2Vec Google News.
    \item \textbf{Query 19} (+0.0433 tanpa stemming/stopword, +0.0413 dengan stopword tanpa stemming): Query ini menunjukkan peningkatan konsisten di dua konfigurasi berbeda. Ini mengindikasikan bahwa topik query memiliki cakupan terminologi yang luas, sehingga sinonim dan istilah terkait yang ditambahkan oleh Word2Vec benar-benar membantu menemukan dokumen relevan yang tidak mengandung term original.
    \item \textbf{Query 37} (+0.0229, tanpa stemming \& stopword): AP naik dari 0.0297 ke 0.0526. Meskipun nilai absolutnya masih rendah, peningkatan relatifnya cukup besar, menunjukkan bahwa expansion membantu sistem menemukan setidaknya beberapa dokumen relevan yang sebelumnya terlewat.
\end{itemize}
Faktor kunci keberhasilan expansion pada query-query ini adalah keberadaan term-term dengan representasi vektor yang kuat dalam Word2Vec (tidak OOV) dan topik query yang cukup luas sehingga variasi terminologi di koleksi dokumen dapat dijangkau oleh term expansion.

\subsection{Query yang Mengalami Penurunan}
Penurunan MAP dapat terjadi ketika:
\begin{itemize}
    \item Term expansion yang ditambahkan tidak relevan dengan konteks query
    \item Query sudah spesifik dan expansion menambahkan noise
    \item Term query tidak ada dalam vocabulary Word2Vec (OOV)
\end{itemize}

Sebagian besar query mengalami penurunan AP setelah query expansion, yang terlihat dari fakta bahwa 3 dari 4 konfigurasi preprocessing menunjukkan penurunan MAP secara keseluruhan. Penurunan ini disebabkan oleh beberapa faktor:
\begin{itemize}
    \item \textbf{Out-of-Vocabulary (OOV)}: Term query yang tidak ada dalam vocabulary Word2Vec Google News tidak dapat menghasilkan expansion, sehingga query tersebut tidak menerima manfaat apapun sementara query lain yang mendapatkan expansion bisa saja menambah noise.
    \item \textbf{Noise dari term expansion}: Beberapa term yang ditambahkan oleh Word2Vec memiliki kemiripan semantik yang terlalu umum atau berada di domain yang berbeda dengan konteks query. Misalnya, kata ``bank'' bisa berarti lembaga keuangan atau tepi sungai---jika query merujuk ke dunia finansial tetapi expansion menambahkan term geografis, maka precision akan turun.
    \item \textbf{Over-expansion}: Menambahkan terlalu banyak term (top-5 per query term) menyebabkan query menjadi terlalu umum dan mengambil dokumen yang tidak relevan, menurunkan precision secara signifikan.
    \item \textbf{Ketidakadaan stemming}: Tanpa stemming, term expansion dan term indeks mungkin berada dalam bentuk morfologis yang berbeda (misalnya ``compute'' vs ``computation''), sehingga pencocokan gagal meskipun secara semantik relevan.
\end{itemize}
Konfigurasi \texttt{stemming=False, stopword=True} mengalami penurunan terbesar (-12.34\%), menunjukkan bahwa tanpa stemming, noise dari variasi morfologis memperburuk efek negatif expansion.

\subsection{Pengaruh Skema Pembobotan}
Sistem mengimplementasikan 7 skema pembobotan:
\begin{itemize}
    \item \textbf{tf\_raw} dan \textbf{tf\_bin} - Sederhana, kurang memperhitungkan pentingnya term
    \item \textbf{tf\_log} dan \textbf{tf\_aug} - Memberikan penalti pada term yang terlalu sering muncul
    \item \textbf{idf} - Hanya mempertimbangkan keunikan term
    \item \textbf{tfidf} dan \textbf{tfidf\_cos} - Menggabungkan TF dan IDF dengan normalisasi
\end{itemize}

Eksperimen dilakukan dengan satu skema pembobotan (\texttt{tfidf\_cos}) untuk membandingkan pengaruh preprocessing dan query expansion secara terisolasi. Dari keempat konfigurasi preprocessing:
\begin{itemize}
    \item \textbf{MAP original terbaik}: 0.1746 (stemming=T, stopword=F)
    \item \textbf{MAP expanded terbaik}: 0.1790 (stemming=T, stopword=T)
\end{itemize}
Hasil ini menunjukkan bahwa untuk retrieval tanpa expansion, mempertahankan stopwords memberikan sedikit keunggulan karena beberapa stopwords dapat berfungsi sebagai konteks (misalnya ``to be or not to be'' sebagai frase spesifik). Namun, untuk query expansion, menghilangkan stopwords menjadi krusial karena stopwords menghasilkan expansion term yang tidak bermakna dan menambah noise. Kombinasi \texttt{tfidf\_cos} dengan stemming dan stopword removal memberikan keseimbangan terbaik antara precision dan recall setelah expansion.

\subsection{Trade-off Recall dan Precision}
Query expansion umumnya meningkatkan recall tetapi dapat menurunkan precision. MAP sebagai metrik yang menggabungkan keduanya memberikan gambaran yang seimbang.

Ya, trade-off ini sangat terlihat pada hasil eksperimen. Pada 3 dari 4 konfigurasi preprocessing, query expansion menurunkan MAP, yang menunjukkan bahwa penurunan precision lebih dominan daripada peningkatan recall. Hanya pada konfigurasi \texttt{stemming=True, stopword=True} peningkatan recall berhasil mengkompensasi penurunan precision, menghasilkan MAP yang lebih tinggi secara keseluruhan. Hal ini menegaskan bahwa:
\begin{itemize}
    \item Query expansion efektif meningkatkan recall (menemukan lebih banyak dokumen relevan melalui variasi terminologi semantik)
    \item Namun tanpa preprocessing yang tepat, term expansion juga menurunkan precision (menaikkan rank dokumen tidak relevan yang kebetulan mengandung term expansion)
    \item Stemming + stopword removal bertindak sebagai filter yang memastikan hanya term bermakna yang diekspansi, sehingga meminimalkan penurunan precision sambil tetap mendapatkan manfaat recall
\end{itemize}

\section{Kesimpulan}
Sistem IR berbasis TF-IDF cosine similarity dengan query expansion Word2Vec telah berhasil diimplementasikan dan dievaluasi pada dataset CISI. Berdasarkan hasil eksperimen, dapat disimpulkan:

\begin{itemize}
    \item Sistem IR berhasil diimplementasikan dengan 7 skema pembobotan TF-IDF dan query expansion berbasis Word2Vec Google News 300 dimensi.
    \item Performa sistem diukur menggunakan MAP pada dataset CISI (1460 dokumen, 76 query dengan qrels) untuk 4 konfigurasi preprocessing dengan skema \texttt{tfidf\_cos}.
    \item MAP original terbaik adalah 0.1746 (stemming=T, stopword=F, tfidf\_cos).
    \item Query expansion hanya meningkatkan MAP pada konfigurasi stemming=True, stopword=True, dari 0.1680 menjadi 0.1790 (peningkatan +6.55\%). Pada tiga konfigurasi lainnya, MAP justru menurun antara 6.93\% hingga 12.34\%.
    \item Tiga query dengan peningkatan AP tertinggi adalah query 111 (+0.0440), 19 (+0.0433), dan 37 (+0.0229), yang menunjukkan bahwa query dengan term yang memiliki representasi Word2Vec yang baik dapat memperoleh manfaat signifikan dari expansion.
    \item Stemming dan stopword removal bersamaan merupakan prerequisite penting untuk query expansion yang efektif, karena stemming menormalkan variasi morfologis dan stopword removal menghilangkan noise dari expansion.
    \item Sistem dilengkapi dengan caching untuk 28 variasi konfigurasi, REST API (FastAPI), dan web interface (React + TypeScript + TailwindCSS).
\end{itemize}
